The HIT takes 35 minutes on average.

This study involves visualizations that show how the computer folder and file organization changes over time.

The file organization works like on most computers, where folders can contain other folders and files.​

The study scenario involves company information where folders are organized to show employee and business files.​

In this example, the top level of the folder is named "All the Employees". This top folder has two sub-folders: one for "John Smith" and one for "Mary Smith".​​

We can also refer to the top folder ("All the Employees") as the parent folder for the "John Smith Folder" and "Mary Smith Folder".

We also say that folders that share the same parent are called siblings, so the "John Smith Folder" and Mary Smith Folder" are sibling folders. In the same way, the folders for "Taxes", "Travel", and "Benefits" are siblings since they are at the same level under the "John Smith Folder".​​

This is the same folder structure with colors added. The same color is used for all the subfolders at the same level.​

Here, we show a different visualization of the same folder organization. Now subfolders are shown inside the parent folder.​

In this version, the key difference is that the height of a rectangle corresponds to the folder's disk space (size on the computer).​

In this example "Taxes" is taller than "Travel", so "Taxes" takes up more space, even though both of them have only one subfolder.​

While height represents file size, there is also space in each folder that is not consumed by a subfolder.​

For example, the "Travel" has a lot of extra space that is not taken by the "Documents" folder.​

This is the space consumed by the files directly in "Travel" folder. This visualization only shows folders and does not show the other files.​

We can also show changes in folder organization over time. ​

Here, we use a separate folder for each month. You can compare the months to see how the folders and subfolders changed from January to February.​

In this image, you see the "Mary Smith Folder" gained a new "Taxes" folder in February.

You also see that the "Mary Smith Folder" is approximately twice the size of the "Taxes" folder, meaning it consumes twice as much disk space.

For the study, we hide folder names, so this is what you will see.​

In this example we show the folder structure of January, February and March.

If a folder shrinks over time, its height will reduce from left to right.

If a folder is deleted, it does not count as a shrinking folder.

2 .  In total: How many folders shrank in size?

The red folder exists in January, but is removed until February. We do not count this folder as a shrinking folder.

The red folder's parent shrinks in size and moves into a newly created folder.

The shrinking node had 2 ancestors in January, which also shrank in size.

The number of folders which shrank is therefore 3.

In this example we show the folder structure of January, February and March.

When we look at the movement of folders, we are interested in those that change their parent.

Subfolders of a moving folder are not counted as moves.

3 .  How many folders move into another folder?

From January to February, the red folder changes its parent.

The white subfolder is not counted as a move between January and February.

From February to March, the white folder moves out of the red folder and gets a new parent.

The number of folders which move is 2.
